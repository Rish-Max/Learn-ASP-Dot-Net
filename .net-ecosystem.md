# .NET Ecosystem — Quick Notes

## .NET Framework

- Original .NET platform (2002)
- Windows-only
- Used for legacy apps (WinForms, WPF, old ASP.NET)
- Maintenance mode (security fixes only)

- Not for new development

---

## .NET Core

- Rebuilt, modern .NET (cross-platform)
- Open source, high performance
- Supported Windows, Linux, macOS
- Last version: .NET Core 3.1

- Name discontinued

---

## .NET (Modern .NET: 5+)

- Unified successor to .NET Core
- Cross-platform and cloud-native
- Active and future-proof
- Current versions: .NET 8 (LTS), .NET 10

- Use this for all new projects

---

## ASP.NET

- Web framework built on .NET
- Used to build Web APIs and web apps
- Handles routing, HTTP, middleware, auth
- Modern ASP.NET runs on .NET (not Framework)

### When to use ASP.NET

- Maintaining or extending **legacy** applications
- Existing apps built on **.NET Framework**
- Windows-only environments

- Not recommended for new projects

---

## ASP.NET Core

- Modern, cross-platform version of ASP.NET
- Built on .NET Core / modern .NET (5+)
- High performance, cloud & Docker friendly
- Supports Minimal APIs, MVC, Razor Pages

### When to use ASP.NET Core

- Building **new Web APIs**
- Microservices and cloud-native apps
- Backend for React / Next.js / Mobile apps
- Docker, Kubernetes, Linux servers
- High-performance and scalable systems
- Default choice for all new development

---

## Summary

- .NET Framework → legacy, Windows-only
- .NET Core → transitional (name retired)
- .NET (5+) → present & future
- ASP.NET → legacy web framework
- ASP.NET Core → modern web framework (use this)
